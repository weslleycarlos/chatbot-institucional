Guia: Rodando Chatbot na Intranet (Sem Permissões de Admin)Este guia foca em como implantar o projeto em ambientes corporativos bloqueados, onde você não pode rodar instaladores .exe ou configurar servidores web complexos.A Estratégia: "Portable Stack"Em vez de instalar o Python no Windows, usaremos uma distribuição portátil. Todo o seu projeto viverá dentro de uma única pasta, que pode ser zipada e enviada para outros computadores.Passo 1: O Python Portátil (WinPython)O WinPython é a solução ideal. Ele é zero-instalação.Baixe o WinPython (versão "dot" ou simples) no Github oficial (procure por WinPython_3.xx).Extraia o arquivo para uma pasta onde você tem permissão (ex: Meus Documentos/GovBot ou C:\Users\SeuUsuario\AppData\Local\GovBot).Dentro da pasta, você verá um executável chamado WinPython Command Prompt.exe. Use apenas ele para rodar comandos.Passo 2: Instalando as Bibliotecas (Localmente)Abra o WinPython Command Prompt.exe e rode:# Isso instala tudo dentro da pasta do WinPython, sem pedir Admin
pip install streamlit langchain-google-genai langchain-community faiss-cpu pypdf python-docx
Passo 3: O Frontend + Backend (Streamlit)Em ambientes restritos, separar Backend (API) e Frontend (React) é complexo. O Streamlit une os dois. Ele gera a interface HTML/CSS automaticamente via Python.Crie um arquivo app.py (código fornecido abaixo).Coloque na mesma pasta do seu projeto.Passo 4: Rodando na Rede (Intranet)Para que outras pessoas na rede acessem seu chat, você não pode rodar apenas como localhost.Descubra o IP da sua máquina (no cmd digite ipconfig, ex: 10.0.0.45).Rode o comando especificando o endereço:streamlit run app.py --server.address=0.0.0.0 --server.port=8501
Agora, qualquer colega na mesma rede pode acessar digitando http://10.0.0.45:8501 no navegador deles.Passo 5: Criando um "Executável" para o UsuárioPara não obrigar ninguém a abrir terminal, crie um arquivo chamado INICIAR_CHAT.bat na pasta raiz:@echo off
echo Iniciando o GovBot...

call "WPy64-31090\scripts\env_for_icons.bat" 
streamlit run app.py --server.address=0.0.0.0
pause
Privacidade e Dados (Offline vs Online)Se a sua intranet tem acesso à internet (apenas via proxy), o código usando Google Gemini funcionará.Se a intranet for totalmente bloqueada (Air Gapped), você precisará trocar a API do Google por um modelo local (LlamaCPP).Para rodar 100% Offline (Opcional):pip install llama-cpp-pythonBaixe um modelo GGUF (ex: llama-3-8b-instruct.Q4_K_M.gguf).No código, troque ChatGoogleGenerativeAI por LlamaCpp.Isso rodará lento (CPU), mas garante sigilo absoluto dos dados.
